# CONTRIBUTING

First of all, thank you for taking interest in ExpWatch.

Although this repository is open source, ExpWatch is also a personal portfolio project maintained exclusively by its author.

Because of that, the development workflow is intentionally simple.

---

## Feature Suggestions

Ideas, feedback, and discussions are always welcome.

If you have:

* Feature requests
* Design suggestions
* Hardware ideas
* UI improvements
* Bug reports

please open an Issue.

Both English and Spanish are accepted.

---

## Pull Requests

Pull Requests are currently not accepted.

All implementation work is performed by the repository owner.

This approach allows the project to remain a personal learning experience while still benefiting from community feedback.

---

## Credits

Community suggestions that become part of ExpWatch will always be credited.

Whenever a feature originates from an external suggestion, the contributor responsible for the idea will be mentioned in the corresponding changelog entry.

---

## Discussions

Please keep all interactions:

* Respectful
* Constructive
* Well-argued

Healthy technical discussions are encouraged.

Harassment, insults, or hostile behavior are not acceptable.

---

## Derivative Projects

You are free to:

* Fork ExpWatch.
* Modify the hardware.
* Create your own versions.
* Build upon the existing design.

Although the MIT License already grants these permissions, mentioning ExpWatch as the original inspiration is greatly appreciated.

---

## Development Workflow

```text
Issue
↓
Discussion
↓
Decision by repository owner
↓
Implementation
↓
Credits in CHANGELOG
```

Thank you for helping improve ExpWatch.
